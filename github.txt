Git Architecture
===================
 
1) Working Tree
 
2) Staging Area
 
3) Local Repo
 
4) Central Repo (Remote repo)
Working Tree → git add → Staging Area → git commit → Local Repo → git push → Remote Repo
# git init : Initialize working tree
 
git status : display working tree status (staged and un-staged)
 
git add : Add file(s) to staging area
    git add <file-name>
    git add .
 
git commit : Commit staged files to local repo
    git commit -m <msg>
 
git push : publish local commits to central repo
 
git restore : to unstage + to discard file changes

.gitignore : it is used to ignore the files which you dont waant to get track/commit.
 
1️⃣ Create .gitignore file
Inside your project folder create a file named:
.gitignore
 
2️⃣ Add files or folders to ignore
Example:
# Ignore log files
*.log
# Ignore compiled files
*.class
# Ignore node modules folder
node_modules/
# Ignore environment files
.env
# Ignore build folder
build/
 
3️⃣ Commit .gitignore
git add .gitignore
git commit -m "added gitignore"
